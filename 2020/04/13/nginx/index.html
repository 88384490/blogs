# Nginx

Nginx作为一款由毛子开发的轻量级服务器，有着便于设置、系统开销小、并发数高等特点。作者由2012年进行开发，2014年开源。

# nginx 常用命令行

nginx // 启动nginx
nginx -c /usr/local/nginx-1.16.1/conf/nginx.conf // 带配置文件启动Nginx
nginx -s reload // 重启nginx
nginx -s stop // 停止nginx，并不保存相关信息
nginx -s quit // 停止ngiinx，并保存相关信息

# nginx 反向代理

代理通常的意义分为正向代理和反向代理，正向代理是客户端访问中转服务器，再由中转服务器携带数据去访问目的服务器，然后将数据转发给客户端。我们通常说的翻墙就是这种模式(喝茶注意，国宝注意)

反向代理是将服务器的数据经由服务器程序转发给客户端，然后再代理客户端的数据转发给服务端，双方都经由中间服务器进行转发来达到互相访问的目的。 面试说法：代理服务器来接收internet上的连接请求，然后将请求转发给网络上的服务器，并将从该服务器得到的结果返回给internet上请求连接的客户端，此时代理服务器对外将表现为一个服务器 正向代理代理的是客户端，反向代理代理的是服务端

server {
    ...
    location / {
        prox_pass http:192.168.2.1:8080; // 设置本地代理地址
        index index.html index.htm;
    }
}

# nginx 负载均衡

将多台服务器以对称的方式组成一个服务器集合，每台服务器都具有等价的地位，都可以单独对外提供服务而无需其他服务器的辅助(去中心化) nginx的负载均衡算法是以轮循的方式按配置的顺序访问每一台服务器 一个域名关联着很多台服务器


    // 设置服务器别名
    upstream myserver {
        // 服务器1
        server 82201230.github.io
        // 服务器2
        server 88384490.github.io
    }

    server {
        ...
        localtion / {
            proxy_pass http://myserver1
        }
    }

每次访问时，nginx会循环调用服务器别名中的服务器，达到分担流量的目的
