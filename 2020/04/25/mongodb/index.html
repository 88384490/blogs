# MongoDB 非关系型数据库

MongoDB是基于分布式文件存储的数据库。与MySQL基于关系型数据管理不同的是，MongoDB是基于非关系型数据来进行数据管理的。MongoDB介于关系型数据库和非关系型数据库之间的产品。
所谓关系型数据管理是指采用了关系模型来组织数据的数据库，其以行和列的形式存储数据，以便于用户理解。
而非关系型数据库则是去掉关系数据库的关系型特性。数据之间无关系，这样就非常容易扩展。 无形之间也在架构的层面上带来了可扩展的能力。NoSQL是一项全新的数据库革命性运动，其拥护者们提倡运用非关系型的数据存储，相对于铺天盖地的关系型数据库运用，这一概念无疑是一种全新的思维的注入。

MongoDB支持的数据结构非常松散，是类似json的bson格式，而json只能存储字符串，因此可以存储比较复杂的查询语句。MongoDB最大的特点就是它支持的查询语句非常强大。　　 其语法类似于面向对象的查询语句，几乎可以实现类似关系型数据库单表查询的绝大部分功能，而且还支持对数据建立索引。

MongoDB的特点:

  • 高扩展
  • 没有复杂型关系
  • 低成本
  • 架构灵活
  • 半结构化数据
  • 分布式存储
  • 多表查询能力较弱
  • 对于node.js的执行效率低

# MongoDB的安装

以下以windows平台做范例

  1. 从官网下载MongoDB安装包
    1.1 在官网首页找到software这一栏，选择community server(社区版)
    1.2 跳转到数据库版本选择后，选择自己的想要下载的数据库版本和系统，以及安装包。一般都是选择(current release)稳定版
    linux的安装方式：官网、MAC的安装方式：官网

  2. 下载完成后，打开安装包，在同意用户协议后来到此界面。我们可以一切按系统默认的路径进行安装，也可以采用自定义安装，自己选择需要 安装的路径。

  3. 设定完安装路径后，设定是使用网络方式运行还是本地方式运行，在这里使用网络方式运行

  4. 这里mongodb是默认安装Mongodb compass的，Mongodb compass是一款以图形化管理mongodb数据的官方管理软件，如果家里网络够快可以选择安装， 如果网络较慢可能会有卡死的情况。

# MongoDB的环境配置

  1. 进入我的电脑->右键属性->高级系统设置->环境变量
  2. 进入环境变量设置窗口后,在path的末尾加入MongoDB所在文件夹下的bin文件夹的路径，以;号结尾。
