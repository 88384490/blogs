# let、const以及变量的解构赋值

  • let
    • let 不允许在同一个作用域内重复声明同一个变量。
    • let 只作用在当前定义的作用域内,如在全局作用域内读取块级作用域内的let变量，则无法读取。
    • 与var不同，无法在声明进行使用，相比var的变量提升问题，会更加严谨。
let name = '1';
let name = '2';
// redeclaration of let name 不能重复声明变量

function foo () {
  let name = 'joe'
}
console.log(name) // name undefined 

console.log(name)
let name = 'joe'
//redeclaration of var name 初始化前无法使用变量

note: 在工作中，我更倾向于使用var来声明全局变量，在函数方法内使用let来声明局部变量。

  • const
    • const通常用来声明常量，const声明的变量必须立刻初始化，声明后的值不能改变
const value = '222';
value = 222;
// invalid assignment to const `value' 常量无法被更改
const value;
// missing = in const declaration 缺少声明

全局属性的问题

var、function声明的全局变量依旧是全局对象的属性
let、const、class声明的全局变量不属于全局对象的属性

let x = 0;
var y = 1

在console中输出
window.x // undefined
window.y // y

当我声明变量后，var变量会成为全局对象，而let变量则不会成为全局对象

  • 结构赋值
    • es6允许按照一定的模式，从数组或对象钟提取值，对变量进行赋值
let [x, y] = [2, 3]
console.log(x) // 2
console.log(y) // 3

let obj = {
  j: 100,
  k: 200
}
let {j, k} = obj
console.log(j, k) // 100, 200

# es6字符串新增方法

  • String.includes() 返回布尔值，表示找到了参数中的字符串
  • String.startsWith() 返回布尔值，表示参数字符串是否在源字符串的开始位置
  • String.endsWith() 返回布尔值，表示参数字符串是否在源字符串的结尾位置
  • 字符串模版 使用``包裹字符串，使用${}包裹变量

# es6箭头函数

  • 箭头函数 === 匿名函数
      // 单参数时，变量不需要加()
  let funcA = value => value;
  console.log(funcA(10)) // 10
  // 多参数时，变量需要加()
  let funcB = (value1, value2) => `变量1为${value1}, 变量2为${value2}`
  // 当函数中内容超过一行时，需要加{}
  let funcC = (value1, value2) => {
    let result = value1 + value2;
    return result;
  }
  // 直接返回一个对象, 因为{}在箭头函数中会被解释为代码块，所以需要在返回中再定义一个对象
  let funcD = () => {
    return {
      value1: 1,
      vlaue2
    }
  }

note1: 箭头函数中没有arguments变量
note2: 箭头函数的this指向问题，箭头函数和匿名函数不同，它没有自己的this，它的this直接指向外部的代码块区域

# es6中的面向对象

  • class关键字
      class Parent () {
    // 构造方法
    constructor (x, y) {
      this.x = x;
      this.y = y
    }
    toString () {
      return `(${this.x}, ${this.y})`
    }
  }

  class Child extend Parent {
    constructor (x, y, z) {
      super(x, y); // 调用父类的constructor
      // 在父类的基础上进行扩展
      this.z = z
    }

    showParam () {
      console.log(this.x, this.y, this.z)
    }
  }

  // 实例化类
  // 子类只是继承父类中的形参， 而不是继承父类实例化后的值
  let parent = new Parent('a', 'b')
  ler child = new Child('a', 'b', 'c')
  // 调用类
  console.log(parent.toString()) // (a, b)
  child.showParam() // a, b, c

class关键字的使用使得javascript能够更好拥有具备面向对象的性质，更方便对类进行继承，逻辑更清晰。
class关键字也广泛的在React中进行使用，因为React专注于页面组件的构建，而当组件需要复用时，可是通过extend继承将需要复用组件中的属性复用到新组件，并在原组件的基础上进行扩展

# es6中的promise异步编程对象 (重点)

  • Promise是异步编程的一种解决方案， 比传统的解决方案，如回调函数和事件，更合理、更强大。
  • 有了Promise对象，就可以将异步操作以同步操作的流程表达出来，避免了嵌套的回调函数。
  • Promise对象提供了统一的接口，使得控制异步操作更加的容易。
// Promise经典使用场景
let promiseObj = new Promise((resolve, reject) => {
  let resp = ajax({
    // 请求参数
  })
  // 逻辑
  if (resp.status === 200) {
    resolve('请求成功')
  } else {
    reject(error)
  }
})
